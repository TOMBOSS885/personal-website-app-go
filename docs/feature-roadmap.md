# 个人网站后续功能优化方向

本文档用于记录 Go 后端 + React 前端个人网站后续可扩展的新功能方向。优先级以实用性、稳定性、安全性和实现收益综合排序。

## 功能方向总览

| 方向 | 功能 | 作用 | 优先级 |
|---|---|---|---|
| 访问体验 | 首页骨架屏 / 图片渐进加载 | 网络较慢时先显示占位内容，背景图、文章封面和项目截图加载更顺滑 | 高 |
| 访问体验 | Live2D 开关与低性能模式 | 移动端或弱性能设备可自动关闭 Live2D、复杂动画和部分装饰效果 | 高 |
| 内容管理 | 草稿箱 / 定时发布文章 | 后台写文章时可保存草稿，并设置未来时间自动发布 | 高 |
| 内容管理 | 文章自动保存 | 防止写文章时刷新、断网或误关闭导致内容丢失 | 高 |
| 内容管理 | 文章标签管理页 | 统一新增、重命名、删除标签，避免文章标签越来越混乱 | 中 |
| 内容管理 | 图片资源库 | 统一管理文章图片、背景图、头像等资源，支持预览、删除和复用 | 高 |
| 音乐模块 | 音乐播放列表排序 | 后台拖拽调整歌曲顺序，前台按排序播放 | 中 |
| 音乐模块 | 歌词显示 | 支持上传 `.lrc` 歌词文件，前台播放器显示同步歌词 | 中 |
| 站点稳定性 | 后台操作日志 | 记录登录、上传、删除、修改文章等操作，方便排查问题 | 高 |
| 站点稳定性 | 数据备份 / 导出 | 后台一键导出文章、项目、主题配置、音乐列表等核心数据 | 高 |
| 站点安全 | 登录失败限制 | 多次密码错误后短暂锁定，降低暴力破解风险 | 高 |
| 站点安全 | 管理后台二次验证 | 可选邮箱验证码或 TOTP，提高后台登录安全性 | 中 |
| SEO | sitemap.xml 自动生成 | 自动输出站点地图，让搜索引擎更容易收录文章和项目页 | 高 |
| SEO | 文章 meta 信息配置 | 每篇文章可设置摘要、关键词、封面和 Open Graph 信息 | 中 |
| 前台互动 | 评论系统 | 支持文章评论，可先做管理员审核模式 | 中 |
| 前台互动 | 留言板 | 独立留言页，适合个人网站访客交流 | 中 |
| 数据统计 | 访问量统计 | 统计页面访问、文章阅读量、来源页面和访问趋势 | 高 |
| 数据统计 | 后台仪表盘增强 | 显示文章数、访问量、热门文章、最近操作和系统状态 | 中 |
| UI 个性化 | 主题预设管理 | 保存多套主题方案，一键切换亮色、暗色、图片背景方案 | 中 |
| UI 个性化 | 前台布局配置 | 后台调整首页模块顺序，如项目、文章、技能、音乐等 | 中 |
| 部署运维 | 后台健康状态页 | 显示数据库、上传目录、磁盘空间、API 状态和版本信息 | 高 |
| 部署运维 | 后台清理缓存 / 无用文件 | 清理未引用图片、旧背景、失败上传残留和临时文件 | 高 |
| 内容展示 | 项目详情页 | 每个项目独立展示截图、技术栈、介绍、链接和更新记录 | 中 |
| 内容展示 | 文章目录导航 | 长文章自动生成目录，提升阅读体验 | 高 |
| 内容展示 | 全站搜索 | 搜索文章、项目、技能等内容，前台和后台都可使用 | 高 |
| 多语言 | 后台多语言编辑增强 | 中英文内容并排编辑，缺失翻译时给出提示 | 中 |
| 移动端 | PWA 离线访问 | 手机可添加到桌面，缓存静态资源和部分文章 | 中 |

## 优先推荐实施

| 排名 | 功能 | 推荐原因 |
|---|---|---|
| 1 | 图片资源库 | 当前系统已经有背景图、文章图、头像、Live2D、音乐等上传能力，统一管理后会明显降低维护成本 |
| 2 | 登录失败限制 | 安全收益高，实现成本相对低，适合优先补齐 |
| 3 | 文章自动保存 + 草稿箱 | 后台写内容时体验提升明显，可减少误操作损失 |
| 4 | 全站搜索 | 内容变多后非常实用，文章、项目、技能都能快速定位 |
| 5 | 健康状态页 + 无用文件清理 | 对 Docker 部署、服务器排错和长期维护很有帮助 |

## 建议实施顺序

### 第一阶段：稳定性和安全

1. 登录失败限制。
2. 后台操作日志。
3. 后台健康状态页。
4. 无用文件扫描与清理。
5. 数据备份 / 导出。

### 第二阶段：内容管理效率

1. 图片资源库。
2. 文章自动保存。
3. 草稿箱。
4. 定时发布。
5. 标签管理页。

### 第三阶段：访问体验

1. 首页骨架屏。
2. 图片渐进加载。
3. Live2D 低性能模式。
4. 文章目录导航。
5. 全站搜索。

### 第四阶段：增强展示和互动

1. 项目详情页。
2. 访问量统计。
3. 后台仪表盘增强。
4. 评论系统。
5. 留言板。

## 可先落地的低风险功能

这些功能改动范围较小，适合作为下一轮优化起点：

| 功能 | 涉及范围 | 风险 |
|---|---|---|
| 登录失败限制 | 后端登录接口、数据库或内存限流 | 低 |
| 文章目录导航 | 前端文章详情页 | 低 |
| 音乐排序 | 后台音乐管理、音乐表排序字段 | 中 |
| 健康状态页 | 后端健康接口、后台新页面 | 低 |
| 图片资源库基础版 | 后台页面、现有上传目录扫描接口 | 中 |

## 后续选择建议

如果优先考虑服务器稳定和安全，建议先做登录失败限制、后台操作日志、健康状态页和无用文件清理。

如果优先考虑使用体验，建议先做图片资源库、文章自动保存、草稿箱和全站搜索。

如果优先考虑前台观感，建议先做首页骨架屏、图片渐进加载、文章目录导航和项目详情页。
